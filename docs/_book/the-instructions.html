3 The Instructions

But back to bookdown …

  1. Install R and RStudio
  2. In the console of RStudio, enter install.packages("bookdown") to install the package.
  3. Create your repo, say myRepo and create an R Project based on that repo.
  4. Create a folder in myRepo called docs, say. Your book is going to live in docs, allowing the rest of your repo to contain an awesome project of some kind.
  5. Create an Rmd file called index.Rmd in docs.

An RMarkdown document begins with some yaml lines. The default that comes with RStudio looks like this:

title: "My Title"
author: "Catherine Dalzell"
date: "2022-09-06"
output: html_document

This yaml instructs knitr to render the document in the usual way. We need to change this if we want to get bookdown output. This simple example works:

title: "A Test Book"
author: "Your Name"
date: "2022-09-06"
site: bookdown::bookdown_site
output: bookdown::gitbook
documentclass: book
link-citations: yes
github-repo: GitHub_username/myRepo
description: "A simple Bookdown demo"

There are other options, but this will get the job done if you want to host on GitHub using GitHub Pages. Note that the repo name is what it says, not myRepo/docs, or wherever you are storing you documentation.

3.1 Document structure

Major headings in markdown are noted with a single hash #. In bookdown, a major heading signifies the start of a new chapter. Double hashes create subsections; these also get links in the table of contents. Triple hashes do not show up in the TOC, but they display in the body of the document as minor headings. It is possible to build a bookdown book from a collection of Rmd files, but the simplest way to get a book is to write everything in one long Rmd file, using major headings to define the chapters.

3.2 knit and publish

  1. click the Knit button in RStudio to render the book. It should display in RStudio’s default browser. You should now see a folder called _book in the docs
  2. When you like what you see and have finished your edits, push the entire project to Github.
  3. Check your repo on GitHub. The files that contain your book will be there, but they will not display as a static web site. We can fix that.
  4. Open the settings of your repo and click on pages. It’s on the left hand side of the page.
